# v0.2.0 - Notes apply workflow and operator launcher

Release artifact: `v0.2.0-notes-apply-asset-setup`.

Release boundary: controlled notes/status/timeline apply workflow, optional asset setup, and a safer local operator launcher. Workbook binaries, generated artifacts, RDF export, SHACL validation, Power Query bridges, and semantic graph publication are not part of this release.

## Added

- Added `Setup Notes Workflow` to create the `Planning Review` notes columns and formula-backed `Decision Staging` / `tblDecisionStaging` table.
- Added the maintained `office-scripts/apply_notes.ts` workflow for two-pass notes/status/timeline writeback: run once to prepare rows, review `Decision Staging`, then run again to apply prepared rows into `Planning Table`.
- Added a live `Planning Review!O1:R3` `ApplyNotes Control` area that shows the last ApplyNotes phase, timestamp, result, and next action.
- Added `Start-AddIn.ps1` and `README_FIRST.md` as the simpler operator entry point for ZIP downloads and workbook-copy use.
- Added npm launcher aliases `start:addin` and `excel:addin`, while keeping `addin:smoke` and `test:smoke` for developer smoke testing.
- Added `office-scripts/apply_asset_mappings.ts` and optional `Setup Asset Workflow` starter surfaces for controlled asset workflow experiments.
- Added public-safe starter TSVs for decision staging and asset setup tables.
- Added `modules/assets.formula.txt` with dynamic-array asset review queues.

## Operator Flow

1. Download and extract the repo ZIP.
2. Run `Start-AddIn.ps1`.
3. Open a workbook copy.
4. Click `Setup + Install + Validate + Outputs`.
5. Click `Copy ApplyNotes Script` and save it in Excel as `ApplyNotes` through `Automate -> New Script`.
6. Type updates in `Planning Review!P:R`, run `ApplyNotes` once to prepare, inspect `Decision Staging`, then run `ApplyNotes` again to apply.

## Deferred

- Workbook binaries or generated workbook artifacts
- RDF/export
- SHACL validation
- Power Query bridge
- Ontology publication
- Production installer or Marketplace package

## Impact

- Workbook behavior: adds a governed notes/status/timeline apply workflow with visible operator state, plus optional asset setup surfaces.
- Operator setup: adds a safer launcher that warns users to work in a workbook copy and opens Excel with the sideloaded add-in.
- Formula logic: existing capital planning formulas are unchanged; new asset formulas are review-only.
- Main report totals: no intended change.
- Subtotal flags: no intended change.
- Cap remaining values: no intended change.
